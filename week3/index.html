      4. From JavaScript: Novice to Ninja, 2nd Edition
        1. Objects: arrays, functions

          An object literal is an object that is created directly in the language by wrapping all its properties and methods in curly braces {}. Object literals are a distinguishing feature of the JavaScript language, as they allow objects to be created quickly without the need for defining a class. They also provide a useful way of organizing your code without polluting the global namespace.

          To create an object literal, simply enter a pair of curly braces: const spiderman = {}; OR const spiderman = new Object(); //former is object literal notation, latter is object constructor (not recommended)

          Accessing properties: dot notation or bracket notation: superman.name OR superman['name']

          • Dot notation is much more common, but bracket notation has a few advantages: it's the only way to access nonstandard property and method names that don’t follow the variable naming rules.

        2. DOM
          1. These methods will return a node object or a node list, which is an array-like object. These objects can then be assigned to a variable and be inspected or modified.
          2. Properties:

            • typeof: ex return, object

            • nodeType property: returns numerical code to signify what type they are

            • nodeName property: returns name of element, ex BODY (uppercase letters)

            • childNodes property returns all the nodes that are children of an element. This will include any text nodes, and since whitespace is treated as a text node, there will often be empty text nodes in this collection.

            • children property only returns any element nodes that are children of that node, so will ignore any text nodes.

            • firstChild property returns the first child of a node

            • lastChild property returns the last child of a node. Be careful when using these properties ― the first or last child node can often be a text node, even if it’s just an empty string generated by some whitespace

            • parentNode property returns the parent node of an element

            • nextSibling property returns the next adjacent node of the same parent

            • previousSibling property returns the previous adjacent node. Once again, these methods find the next and previous node, not element, so they will often return a blank text node

            • innerHTML element property returns all the child elements of an element as a string of HTML. This is also used to create new text node, replacing existing child elements.

            • Updating CSS

            • style property: superman.style.border = "red 2px solid";

            • camelCase notation: background-color is backgroundColor

            • can also use bracket notation (no need of camelCase notation): superman.style['background color'] = 'blue';

            • display property: superman.style.display = 'none'; superman.style.display = 'block';

            • getComputedStyle() that will retrieve all the style information of an element that is given as a parameter (only applies to inline styles and styles set using JS): getComputedStyle(superman);

            • CSSStyleDeclaration objects have some built-in methods to help extract the information

            • to find out the element's color property: getComputedStyle(superman).getPropertyCSSValue('color').cssText;

            • getPropertyCSSValue() is a method of CSSStyleDeclaration

          3. Methods:

            • Document.body returns the body element of a web page

            • Document.images returns a node list of all the images contained in the document.

            • Document.links returns a node list of all the elements and elements that have an href attribute.

            • Document.anchors returns a node list of all the elements that have a name attribute.

            • Document.forms returns a node list of all the forms in the document. This will be used when we cover forms in Chapter 8.

            Node lists

            • can be accessed using index notation

            • has length property, which can be used to iterate through every element using a for loop

            • don’t have any other array methods such as slice, splice and join

            • convert node list into an array using either Array.from(): const imageArray = Array.from(document.images); or the spread operator: const imageArray = [...document.images];

            • To find the value of a node, use either nodeValue method or textContent property

            • getAttribute() method returns the value of the attribute provided as an argument

            • setAttribute can change the value of an element’s attributes. It takes two arguments: the attribute that you wish to change, and the new value of that attribute: wonderWoman.setAttribute('class', 'villain');

            • To access attributes, use either dot notation getAttribute()

            • To modify class names of an element, use setAttribute() method or className property: wonderWoman.className; We can change the class back to 'hero' with the following code: wonderWoman.className = 'hero'. Changing the className property of an element by assignment will overwrite all other classes that have already been set on the element. This problem can be avoided by using the classList property instead.

            • classList property is a list of all the classes an element has

            • add method can be used to add a class to an element without overwriting any classes that already exist: wonderWoman.classList.add('warrior');

            • remove method will remove a specific class from an element: wonderWoman.classList.remove('warrior');

            • toggle method is a particularly useful method that will add a class if an element doesn’t have it already, and remove the class if it does have it. It returns true if the class was added and false if it was removed:

            wonderWoman.classList.toggle('hero'); // will remove the 'hero' class

            class back << true

            • To create a dynamic markup:

            1. Create the element node. createElement() method that takes a tag name as a parameter and returns that element: const flash = document.createElement('li');
            2. Create the text node. document.createTextNode() method takes a parameter, which is a string containing the text that goes in the node: const flashText = document.createTextNode('Flash');
            3. Append the text node to the element node. Use appendChild() method that will add another node (given as an argument) as a child node (bottom of the list): flash.appendChild(flashText); or textContent property: const flash = document.createElement('li'); flash.textContent = 'Flash';

            • insertBefore() method will place a new element before another element in the markup: heroes.insertBefore(aquaman,wonderWoman);

            • removeChild() method: heroes.removeChild(aquaman);

            • replaceChild(): h1.replaceChild(newText,oldText);

          4. Functions

            • To support older versions of Internet Explorer, you could create a function that will add extra class to an element, rather than just replace the current class: function addClass(element,newClass){}

            • A Function To Create Elements: function createElement (tag,text) {}

          5. Getting and Setting Attributes

            • getElementById() is a very quick way of finding elements in a document and is probably the most commonly used method of accessing elements.

            • getElementsByTagName() returns a live node list of all the elements with the tag name that is provided as an argument :const listItems = document.getElementsByTagName('li'); // Live collections

            • getElementsByClassName() will return a live node list of all elements that have the class name that is supplied as an argument: const heroes = document.getElementsByClassName('hero'); // Live collections

            • document.querySelector() allows you to use CSS notation to find the first element in the document that matches a CSS selector provided as an argument. If no elements match, it will return null: document.querySelector('#bats');

            • document.querySelectorAll() method also uses CSS notation but returns a node list of all the elements in the document that match the CSS query selector (not a live node list though): document.querySelectorAll('.hero');
