      1. CSS3 Transforms and Transitions
        1. Transforms
          1. Transforms require vendor prefixing for IE9, Android up to 4.4.3, iOS8, and Blackberry 10
            1. -webkit-transform: translate(45px,-45px); /* iOS8, Android ↵4.4.3, BB10 */
            2. -ms-transform: translate(45px,-45px); /* IE9 only */
          2. Transforms don’t work on inline elements. But that’s easy enough to fix. We’ll just add display: inline-block; to the element
          3. Cascades for moltiple transformations: transform - add space, order of transform functions does matter: translateX(40px) scale(1.5);
          4. transform-origin property
            1. has the same syntax as the background-position property
            2. prefixing being required when the transform property requires it
          5. Transform functions
            1. translate
              1. translate(x,y)
              2. translatex
              3. translatey
              4. no impact on the document flow
              5. alternative of the behavior of position: relative;
            2. rotate
              1. rotate()
              2. degrees, grads, radians, or turns
            3. scale
              1. scale(x,y)
              2. scaleX(x)
              3. scaleX(y)
              4. If only one value is provided, it will be used for both the x and y values, growing or shrinking your element or pseudo-element while maintaining the original aspect ratio.
              5. no impact on the document flow
              6. alternative of altering width, height, or font-size
            4. skew
              1. skew(x,y)
              2. skewX()
              3. skewY()
              4. If the second parameter is omitted, the skew will only occur on the x axis
        2. Transitions
          1. Steps:
            1. PRE-STATE: Declare the original state of the element in the default style declaration.
            2. POST-STATE: Declare the final state of your transitioned element; for example, a :hover state.
            3. IMPORTANT: Include the transition functions in your default style declaration using the transition properties:
              1. transition-property
                1. -webkit- prefix for all of your transition properties
                  1. -webkit-transition-property: -webkit-transform;
                    transition-property: transform;
                2. In itself, the transition-property property has no effect; that’s because we still need to specify the duration of the transition
                3. Multiple declarations separated by commas or use the keyword "all"
                  1. transition-property: transform, color;
              2. transition-duration
                1. transition-duration: 0.2s;
                2. 200ms is generally considered the optimum time for a transition: anything slower will make the website seem slow, drawing generally unwanted attention to what was supposed to be a subtle effect. Anything faster may be too subtle.
              3. transition-timing-function
                1. ease, linear, ease-in, ease-out, or ease-in-out
                2. cubic-bezier
                3. steps(n, start) or steps(n, end)
              4. transition-delay
                1. -webkit-transition-delay: 50ms;
                  transition-delay: 50ms;
                2. negative delays - it will start partway through the animation. On the way back, it will jump 75% of the way through the transition, and then transition back to the default state.
          2. makes transforms dynamic
          3. -webkit- vendor prefix is still needed for older mobile devices
          4. Any property changing from one value to another for which you can find a valid midpoint can be transitioned.
          5. SHORTHAND: transition: transform 0.2s ease-out 50ms;
            1. Note that the properties in the shorthand syntax can be in any order, however, if a delay is included, you must also include a duration, and the duration must precede the delay.
          6. Multiple Transitions in one call
            1. If you want your properties to transition at different rates, or if you just want a select few to have a transition effect, include them as a comma-separated list:
              transition-property: transform, color;
              transition-duration: 0.2s, 0.1s;
              transition-timing-function: ease-out, linear;
              transition-delay: 50ms;
            2. SHORTHAND: transition: transform 0.2s ease-out 50ms, color 0.1s linear 50ms;
          7. How do we know when a transition has ended? A transitionend event—prefixed and camel-cased as webkitTransitionEnd for older mobile browsers—is fired upon completion of a CSS transition in both directions.
          8. Animations - keyframes
            1. -webkit- still required
            2. Animation Properties
              1. animation-name
                1. Note that you should not put quotes around the animation name in either the property value or the @keyframes at-rule, as the specifications state the name is an identifier and not a string, so browsers don’t support quoted animation names.
              2. animation-duration
              3. animation-timing-function
                1. ease, linear, ease-in, ease-out, ease-in-out
                2. cubic-bezier()
                3. step-start, step-end
                4. steps(number, direction)
              4. animation-iteration-count
              5. animation-direction
              6. animation-delay
              7. animation-fill-mode
              8. animation-play-state
              9. SHORTHAND: animation: 300ms ease-in alternate 5s backwards appearDisappear;
              10. Nultiple Animations animation:
                animationOne 300ms ease-in backwards,
                animationTwo 600ms ease-out 1s forwards;
