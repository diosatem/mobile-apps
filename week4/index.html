    2. Insights:
      1. Forms
        1. The legacy DOM had a useful property called document.forms that returns an HTML collection of all the forms in the document in the order they appear in the markup.
        2. square bracket notation: const form = document.forms['search']; (this is also required if the form’s name attribute contains any invalid characters, such as spaces or dashes)
        3. index notation: const form = document.forms[0];
      2. Modern JavaScript Development
        1. Libraries
          1. A JavaScript library is a piece of code that provides several methods that make it easier to achieve common tasks.
          2. DOM API
          3. jQuery
            1. the most popular of all the JavaScript libraries used today
            2. has a plugin system that makes it easy to extend and use to build common web page elements, such as a lightbox or carousel widget.
          4. Underscore and Lodash
            1. They both provide a large number of utility functions under the namespace _ (this is where they get their name from as the _ character is often referred to as an 'underscore' or a 'lodash').
          5. Vanilla JS
            1. showcases plain JavaScript as if it were a library, highlighting that many tasks can be accomplished with a similar amount of code but much better performance. This is now truer than ever since ES6, as a lot of problems that some libraries addressed have been fixed in the vanilla flavor of JavaScript.
          6. Advantages
            1. Using a library means you can be confident that your code will be as bullet-proof as possible in many browsers.
            2. In addition, there will usually be lots of online documentation and a strong community ready to help out if you become stuck.
          7. Disadvantages
            1. You need to include the code for the library as well as your own code. This increases the amount of code that needs to be downloaded by a website, which in some cases can cause performance issues.
            2. Another problem with libraries is that they might fail to implement the functionality in the precise way that you want it to perform.
        2. Modular JavaScript
          1. A module is a self-contained piece of code that provides functions and methods that can then be used in other files and by other modules.
          2. A ES6 module file is just a normal JavaScript file, but uses the keyword export to specify any values or functions that are to be made available from the module.
